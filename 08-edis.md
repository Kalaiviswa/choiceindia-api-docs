# eDIS (CDSL)

Electronic Delivery Instruction Slip verification for non-POA clients, via CDSL.

### API for DIS verification from CDSL

```
POST /api/OpenAPI/VerifyDIS
```

**Operation ID:** `OpenAPI_VerifyDIS`

**Description**

```text
Description

    This API use to fetch DIS verification from CDSL.

Parameters

    BOID: Client BO ID (Beneficiary Owner Identification Number of Demat account, provided by Depository)
    
    CtrBOID: Counter BO ID
    
    ExId: Segment Id (Only NSE/BSE CASH applicable)
    
    channel: 0 for WEB, 1 for MOB
    
    ISIN: Scrip ISIN
    
    Quantity: Quantity to EDIS
    
Parameter Types

    1."string" stands for String Type
    2.0 (Zero) stands for Numeric.
    
Segment Master ID's

    NSE-CASH: 1
    
    BSE-CASH: 3
```

**Consumes:** `application/json`, `text/json`, `application/xml`, `text/xml`, `application/x-www-form-urlencoded`  
**Produces:** `application/json`, `text/json`, `application/xml`, `text/xml`

**Request Body**

Schema: **`VerifyDISOPIReq`**

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `BOID` | string | No |  |
| `CtrBOID` | string | No |  |
| `ExId` | integer (int32) | No |  |
| `channel` | integer (int32) | No |  |
| `eDISStocks` | array of `EDISStock` | No |  |

**`EDISStock`**

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `ISIN` | string | No |  |
| `Quantity` | integer (int32) | No |  |


**Responses**

| Code | Description | Schema |
| --- | --- | --- |
| 200 | Sucess Response | `ResponseMessage` |
| 401 | Unauthorized, Session expired or does'nt exists | `string` |
